Faculty Excel Upload & Result Generation

Faculty Member Core Feature

Scenario Overview

A faculty member logs into SRAC to process student examination results. They upload an Excel file containing student marks, and SRAC automatically validates the format, analyzes performance metrics including SGPA, percentage calculations, and fail counts. The system then generates comprehensive reports with visualizations that can be exported as professional PDFs for department sharing.

Automated Excel validation
Real-time SGPA calculation
Professional PDF export

License Key Verification System

Admin/Faculty Security Feature

Scenario Overview

When SRAC launches, it automatically verifies the institution's license key with the remote server. The system checks the College ID and authentication key to ensure authorized usage. If verification fails due to expired license, invalid key, or unauthorized access attempt, the system denies access and displays appropriate error messages.

Remote server validation
College ID verification
Secure access control

Multi-Class Batch Processing

Faculty/Admin Advanced Feature

Scenario Overview

Faculty members can process multiple Excel files simultaneously for different class sections (A, B, C, etc.). SRAC handles batch processing by validating each file, analyzing performance metrics for individual sections, and generating both section-wise detailed reports and a consolidated summary report comparing all sections.

Multiple file upload
Section-wise analysis
Consolidated reporting

User Feedback Collection

All Users Enhancement Feature

Scenario Overview

After completing result analysis, SRAC displays an optional feedback form to collect user insights. Students and faculty can provide ratings, comments on result clarity, visualization quality, and suggestions for improvements. This feedback is logged and analyzed to drive continuous system enhancement.

Optional feedback popup
Rating and comments
Improvement tracking

Unauthorized Access Prevention

Unauthorized User Security Feature

Scenario Overview

When someone attempts to run SRAC on an unauthorized system without a valid license key, the security system activates. SRAC checks for valid credentials with the remote server, and upon finding none or detecting invalid access, immediately denies entry and displays security warnings before safely exiting.

Security validation
Access prevention
Safe system exit
